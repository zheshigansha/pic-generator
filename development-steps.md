# VisionFit Pro — 开发步骤分解

**版本**：v0.1
**日期**：2026-06-20

---

## 第一阶段：技术方案确定（P0）

### Step 1.1：选择AI图片生成API

需要确定的选项：
- **Stable Diffusion**（可自部署，开源，免费但需GPU资源）
- **FLUX**（效果好，API调用）
- **DALL-E 3**（OpenAI官方，图质好，按调用收费）
- **Midjourney**（效果好，但API不支持，需网页接）

| 方案 | 优点 | 缺点 |
|------|------|------|
| Stable Diffusion | 免费、可自部署、无限生成 | 需要GPU服务器、运维成本 |
| FLUX | 效果好、API调用简单 | 按次收费 |
| DALL-E 3 | 质量稳定、微软azure稳定 | 成本较高 |
| Midjourney | 效果好 | 不能API、需手动操作 |

**推荐**：先用 **FLUX** 或 **DALL-E 3** API 做MVP，按调用量收费，不需要 GPU。

### Step 1.2：确定技术栈

| 层级 | 选项 |
|------|------|
| 前端 | React / Next.js（主流） / Vue |
| 后端 | Node.js / Python FastAPI |
| 数据库 | PostgreSQL（存用户、账号关系）/ SQLite（轻量MVP） |
| 存储 | AWS S3 / 阿里云OSS（存图片）/ 本地存储（MVP） |
| 社媒API | Facebook Graph API |

**推荐MVP技术栈**：
- 前端：Next.js（自带页面路由，开发快）
- 后端：Python FastAPI（AI调用方便）
- 图片存储：本地文件夹（先跑通）
- 数据库：先不做（用文件管理代替）

### Step 1.3：确定社媒接入方式

Facebook Graph API 接入需要：
1. 客户授权运营方作为 App 管理员
2. 申请 Facebook App 审核（需要公司资质）
3. 获取 Page Access Token

**注意**：Facebook 审核比较严格，建议先不用做发布功能，先跑通"上传 → 生成 → 审核"这个闭环。

## 第二阶段：核心功能开发（P0）

### Step 2.1：图片上传功能

**功能描述**：
- 运营方上传衣服图片（支持 JPG/PNG/WEBP）
- 支持一次上传多张
- 显示上传进度

**技术要点**：
- 前端拖拽上传
- 后端接收并保存图片到本地/S3

**验收标准**：
- 上传一张衣服图片能成功保存
- 图片可以在网页上预览显示

---

### Step 2.2：AI生成模特图功能

**功能描述**：
- 上传衣服图片后，点击生成
- AI 根据衣服图生成穿上该衣服的模特效果图
- 一次生成 2-4 张不同模特/场景的图，供选择

**技术要点**：
- 调用 FLUX/DALL-E API
- 设计合适的 Prompt（提示词），让生成的模特穿上指定衣服
- 可能需要"衣服图作为参考"的生成模式（img2img）

**验收标准**：
- 传入一张衣服图，能生成模特穿这件衣服的效果图
- 生成结果能明显看出是同一件衣服

---

### Step 2.3：审核界面

**功能描述**：
- 显示生成的 2-4 张图
- 运营方选择"通过"或"重生成"
- 通过的图进入待发布队列

**技术要点**：
- 图片展示：网格布局，点击放大预览
- 状态标记：待审核 / 通过 / 重生成

**验收标准**：
- 能看到所有生成的图
- 能选择其中一张标记为"通过"
- 能对不满意的图发起重生成

---

### Step 2.4：（扩展）社媒发布功能

**功能描述**：
- 将审核通过的图，发布到客户 Facebook/Instagram 账号
- 支持填写文案
- 显示发布状态

**技术要点**：
- Facebook Graph API 接入
- 图片上传到 Facebook 相册
- 文本发布到 Feed

**验收标准**：
- 能选择图片和文案
- 能发布到指定的 Facebook 主页
- 能在 Instagram Business 账号看到发布内容

---

## 第三阶段：测试与调优（P0）

### Step 3.1：生成质量测试

**测试内容**：
- 不同衣服类型（T恤、连衣裙、外套、裤子）生成效果
- 不同肤色模特（白肤色、深肤色）生成效果
- 场景多样性（室内、室外、街拍）

**验收标准**：
- 生成结果看起来真实、自然
- 衣服特征（颜色、图案）在模特身上清晰可辨认

---

### Step 3.2：提示词调优

**工作内容**：
- 根据测试结果，调整 AI 提示词模板
- 针对不同衣服类型设计不同提示词策略

---

## 第四阶段：小范围试运营（P0）

### Step 4.1：真实客户测试

**工作内容**：
- 用 1-2 个真实客户测试完整流程
- 收集客户反馈：图满不满意、发布顺不顺畅

---

## 第五阶段：迭代优化（P1/P2）

### 扩展功能（按优先级）

**P1（第二阶段后可加）**：
1. **素材管理**：历史图片整理，按客户分类
2. **文案辅助生成**：AI 根据图片生成配套文案
3. **多账号切换**：管理多个客户的社媒账号

**P2（后续迭代）**：
1. **素材接收箱**：给客户建独立上传页面
2. **定时发布**：预设发布时间
3. **更多平台**：Pinterest、TikTok

---

## 开发顺序总览

```
第1步：选AI生成API + 搭基础框架
第2步：做图片上传
第3步：接AI生成API，做生成功能
第4步：做审核界面
第5步：（扩展）接Facebook API，做发布功能
第6步：测试调优
第7步：试运营
第8步：迭代
```
